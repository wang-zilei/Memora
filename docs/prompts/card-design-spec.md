# 意图卡片设计规范

> 10 个意图大类的卡片输出格式与叙事风格定义。
> 核心约束：每张卡片 = 标题 + 主体，叙事需体现多轮对话的追问、修正与递进。

---

## 统一规则

所有卡片共用以下字段：

```json
{
  "title": "标题（5-15字，复习弹窗直接弹标题即可）",
  "card_type": "意图大类中文值：概念理解/事实查询/技能学习/操作指南/内容创作/文本处理/规划决策/头脑风暴/交互陪伴/其他",
  "original_question": "用户原始问题（1句）",
  "narrative": "卡片主体：按该意图规定的叙事风格撰写",
  "tags": ["2-5个标签，支持层级如 编程/Python"]
}
```

- 用户 = **"你"**
- AI = **模型平台名称**（豆包、GPT、Claude 等）
- 叙事有温度，像朋友在讲述你的经历
- **多轮对话的递进必须体现**：追问、修正、补充、认知变化，是叙事的重要组成部分，不要只给最终结论
- content_creation 和 text_processing 额外有 `full_output` 字段存完整内容

## 1. 概念理解

**核心**：从多轮对话中识别概念边界与认知突破点，自主选择最合适的组织结构，输出用户的"认知增量"。**多轮对话中的追问、修正、补充和认知递进，必须体现在叙事中。**

**主体格式**：由 LLM 根据对话特点自主选择组织结构，不强制固定段数。

- **组织结构选择**：
  - **金字塔型**：一句话定义 → 逐层展开关键属性与细节。适合"XX 是什么"类。
  - **3W1H/因果链型**：从现象/问题出发 → 推导原因 → 给出可操作结论。适合"为什么 XX 会这样"类。
  - **对比型**：点明各自本质 → 列出关键差异。适合"XX 和 YY 的区别"类。
  - **演进型**：过去以为 → 实际上 → 新理解。适合用户有明显认知转折的多轮追问。

- **叙事规则**：
  - 第一段必须是**定义段**：一句话点明概念本质，不用比喻，50 字以内。
  - 后续为**展开段**：讲清原理、关键要素、适用场景，体现"你获得了什么新理解"。
  - **追问与递进必须体现**：如果对话中有追问、修正、补充，必须写进叙事。不要只给结论——读者需要知道"你是怎么一步步搞懂的"。
  - 人称：用户="你"，AI=平台名称（豆包/GPT/Claude等）。
  - 字数：300-600 字。段落之间用换行符分隔，不使用 [定义][机制] 等标签。

**示例**（金字塔型）：
> Python的GIL（全局解释器锁）是CPython解释器中的一个互斥锁机制，它确保同一时刻只有一个线程执行Python字节码。
> 你以前以为多线程就是多核并行，和豆包讨论后发现，GIL的存在是因为CPython的内存管理（引用计数）不是线程安全的——如果允许多线程同时操作引用计数，会导致内存泄漏或崩溃。
> 但你接着问了一个关键问题：那多线程在Python里还有什么用？豆包解释后你意识到，GIL只在CPU密集操作时生效，I/O密集型任务（网络请求、文件读写）在等待I/O时会自动释放GIL，所以多线程在这类场景下仍然有效。
> 最终的认知突破点是：GIL不是Python的'缺陷'，而是CPython在引用计数内存管理下的必然产物。你学到的结论是：CPU密集用multiprocessing，I/O密集用threading或asyncio。

## 2. 事实查询

**核心**：极简与高信噪比，用户只需要最终答案。

**主体格式**（一段）：

- 直接给出答案，附带关键补充（如时间、来源、相关实体）

**示例**：
> 《活着》的作者是余华（1960-），中国当代作家。这部小说首次发表于1992年，讲述了一个普通农民福贵在20世纪中国社会巨变中经历的一系列苦难与失去。余华凭此书奠定了在中国当代文学中的地位。

## 3. 技能学习

**核心**：将散落的教学对话重组为具有递进逻辑的学习路径。**追问、练习反馈、认知修正必须体现在叙事中。**

**主体格式**（3-4段）：

- **[目标]** 学什么、要达到什么水平
- **[路径]** 按先后顺序拆解的学习步骤（1. 2. 3. ...），每个步骤独占一行
- **[避坑]** 常见错误和注意事项
- **[追问与进阶]**（可选）用户追问或进阶问题

**示例**：
> 你的目标是用Python的asyncio编写异步爬虫，能在不阻塞的情况下并发抓取多个网页。
> 1. 理解async/await基本语法
> 2. 学会用asyncio.gather并发执行任务
> 3. 用aiohttp替代requests做异步请求
> 4. 用信号量Semaphore控制并发数量
> 5. 为每个请求添加异常处理和超时控制。
> 常见坑：不要混用同步和异步代码（会阻塞事件循环）；并发数不要太大（目标站点会封IP）；记得处理超时和异常情况。
> 你接着问了一个问题：那如果目标站点有反爬虫机制怎么办？豆包教你用随机延迟和User-Agent轮换应对，你决定把这个也纳入你的爬虫脚本。

## 4. 操作指南

**核心**：强执行导向，步骤严密、可直接操作。**追问、排查过程、方案调整必须体现在叙事中。**

**主体格式**（2-3段）：

- **[步骤]** 按序号列出操作步骤（1. 2. 3. ...，每个步骤独占一行，含关键命令/代码）
- **[注意]** 环境依赖、预期结果、常见报错
- **[追问与修正]**（可选）报错排查、方案调整过程

**示例**：
> 1. 安装依赖：`pip install aiohttp aiofiles`
> 2. 创建异步函数：`async def fetch(url)`
> 3. 用`asyncio.Semaphore(10)`限制并发
> 4. 用`asyncio.gather(*tasks)`并发执行
> 5. 将结果用aiofiles异步写入文件。
> 需要Python 3.7+。预期结果：10个网页在2-3秒内完成抓取。注意不要用过大的并发数，否则会被目标站点封禁。如果报`Connection refused`错误，检查目标URL是否可达。
> 你运行脚本后发现部分请求超时，和豆包讨论后为每个任务加了`asyncio.wait_for(task, timeout=10)`的超时控制，问题解决了。

## 5. 内容创作

**核心**：提取最终创作成果及关键参数，舍弃中间草稿。

**主体格式**（两段）：

- **[full_output]** 最终版完整内容（不压缩，单独字段）
- **[narrative]** 创作背景：主题、受众、基调、迭代说明（不重复 full_output 内容）

**示例**：
> （full_output：完整的文案/代码/方案全文，保持原样不压缩）
> 这是一份面向技术总监的Q1技术债务汇报PPT大纲，基调偏正式。经过三轮修改，最终版本删减了冗长的技术细节，增加了ROI分析部分。

## 6. 文本处理

**核心**：呈现处理动作与处理结果。

**主体格式**（两段）：

- **[full_output]** 处理后的最终文本（不压缩，单独字段）
- **[narrative]** 处理说明：做了什么处理、关键改动方向（不重复 full_output 内容）

**示例**：
> 将一段约500字的中文技术文档翻译成了英文，目标是保持技术术语的准确性，同时让英文表达自然流畅。
> （full_output：完整的英文译文）。关键术语处理："中间件"译为"middleware"而非直译；对原文中模糊的表述根据上下文补全了具体含义。

## 7. 规划决策

**核心**：结构化对比，帮助用户快速回顾权衡过程与结论。**追问、方案调整、权衡变化必须体现在叙事中。**

**主体格式**（2-3段）：

- **[分析]** 对比的方案、评估维度、关键发现
- **[结论]** 最终推荐/决策及理由
- **[追问与修正]**（可选）决策追问、方案调整过程

**示例**：
> 你比较了三种Python爬虫并发方案：多线程（threading）、多进程（multiprocessing）、异步IO（asyncio）。评估维度包括：代码复杂度、性能、稳定性和资源占用。多线程实现最简单但受GIL限制无法真正并行；多进程能真正并行但内存开销大；异步IO在I/O密集型场景性能最优且资源占用低。
> 结论：对于爬虫场景（典型的I/O密集），推荐使用asyncio + aiohttp，配合Semaphore控制并发数。这是性能与复杂度的最优平衡点。

## 8. 头脑风暴

**核心**：发散性信息的聚合与分类。**追问、方向调整、优先级变化必须体现在叙事中。**

**主体格式**（2-3段）：

- **[议题]** 讨论的核心问题
- **[方向]** 按主题分组整理的创意列表（每组2-4条，每个方向独占一行）
- **[追问与调整]**（可选）方向调整、优先级变化过程

**示例**：
> 你在讨论如何提升个人知识库产品的用户粘性。
> 方向一（游戏化）：连续使用天数（streak）展示、知识积累进度条。
> 方向二（社交）：分享卡片为图片、匿名浏览他人公开知识库。
> 方向三（智能推荐）：基于已有卡片推荐相关话题、定期复习激活遗忘知识。
> 你接着问：这些方向中哪个ROI最高？和豆包讨论后确认，游戏化的实现成本最低且效果最直接，优先落地。社交方向虽然效果好但涉及隐私处理，需要谨慎设计。

## 9. 交互陪伴

**核心**：情绪价值的沉淀，侧重心理感受和对话氛围。**情绪变化、对话转向、体验升级必须体现在叙事中。**

**主体格式**（一段或多段）：

- 按背景→互动→结果写成叙事，多轮交互中的变化必须体现

**示例**：
> 你最近在用英语练口语，心情有些低落，于是和豆包聊了一会儿。你们模拟了几轮日常对话场景，它帮你纠正了几个常见语法错误，还推荐了一些实用的表达方式。
> 你接着问："能帮我模拟一下压力面试吗？"豆包切换了模式，问了几个更有挑战性的问题。这几轮问答让你出了一身汗，但也让你意识到自己需要更多练习。
> 练习结束时你感觉放松了不少，也记住了几个新的句型。你决定下周再做一次模拟。

## 10. 其他

**核心**：兜底逻辑，保留最基础的历史记录。

**主体格式**（一段）：

- 极简摘要，去噪压缩

**示例**：
> 你和GPT聊了几句关于天气和周末计划的话题，没有实质性的知识产出。

## 叙事长度约束

| 类型 | narrative 字数范围 |
|------|------------------|
| 概念理解 | 300-600 字 |
| 事实查询 | 50-100 字 |
| 技能学习 | 350-500 字 |
| 操作指南 | 250-400 字 |
| 内容创作 | 产出全文（narrative 50-100 字） |
| 文本处理 | 原文全文（narrative 50-100 字） |
| 规划决策 | 300-500 字 |
| 头脑风暴 | 300-500 字 |
| 交互陪伴 | 150-300 字 |
| 其他 | 30-80 字 |
