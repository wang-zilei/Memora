# 产品思路拓展方法

> 记录用于优化产品时拓展思路的方法和框架

---

## 一、多模型交叉提问

把同一个产品问题丢给 Claude、GPT-4、Gemini、DeepSeek，对比它们的回答。

**为什么有效：** 每个模型有不同的训练数据偏置和推理风格——有的偏商业视角，有的偏技术视角，有的擅长发现你忽略的边缘场景。

### 进阶用法

- **角色扮演提问：** "你是抖音产品经理，你会怎么优化这个产品的用户增长？"
- **攻击性提问：** "如果这个产品注定失败，最可能的 3 个原因是什么？"（pre-mortem 分析法）
- **极端场景提问：** "如果用户一天要沉淀 50 段对话，这个产品哪里会崩？"

## 二、框架驱动思考

| 框架 | 适合的场景 | 你的产品可以怎么用 |
|------|-----------|-------------------|
| **Jobs to Be Done** | 理解用户"雇佣"产品的真实任务 | 用户不是要"知识库"，是要"不再忘记重要的对话洞见" |
| **Hook Model** (触发→行动→奖励→投入) | 设计用户习惯 | 抓取后给用户的"奖励"是什么？怎么让用户想再回来？ |
| **北境指标法** | 找到最核心的增长指标 | 不是"注册用户数"，而是"成功沉淀的对话段数" |
| **RICE 优先级** | 功能排序 | Reach × Impact × Confidence ÷ Effort |
| **Five Whys** | 深挖问题根因 | 为什么用户不手动整理对话？→ 因为太麻烦 → 为什么麻烦？… |

## 三、横向竞品拆解

### 参考项目

已有的三个参考仓库：
- `ctxport/` — 多平台对话复制/导出
- `chat-export/` — ChatGPT / Claude / Gemini 导出扩展
- `gemini-voyager/` — Gemini 增强与导出扩展

### 方法

- **画竞品矩阵：** X 轴 = 自动化程度（手动→全自动），Y 轴 = 知识组织能力（原始文本→结构化卡片→知识图谱），看你的产品落在哪
- **拆解每个竞品的"一个亮点"：** 不是全面对比，而是每个竞品找 1 个你最想偷过来的设计决策
- **看非直接竞品：** Notion AI、Mem.ai、Obsidian、Readwise — 它们在做类似"信息沉淀"的事，但入口不同

## 四、用户旅程地图

把用户从"第一次听说这个产品"到"用了一周后"的每一步画出来：

```
发现 → 安装扩展 → 安装桌面应用 → 第一次抓取 → 看到第一张卡片 →
搜索 → 第二次使用 → 一周后还记得回来
```

在每个节点问：
- 用户在这里的**心理活动**是什么？
- **最容易流失**的摩擦点在哪里？
- 有没有一个**aha moment**（"哇这个好用"的瞬间）？

---

## 五、数据驱动的"意外发现"

产品跑起来之后，最有价值的思路往往来自数据里的异常：

- 用户抓了但**从来不搜索** → 卡片质量有问题？还是不需要搜索？
- 某个平台抓取**失败率特别高** → 是不是该优先支持？
- 用户**手动编辑了卡片** → 说明自动总结哪里不准？
- 用户**导出了某些卡片但没导出另一些** → 什么内容对用户真正有价值？

---

## 推荐组合（当前阶段：MVP 打磨）

1. **多模型 pre-mortem 提问**（"这个产品为什么会失败"）— 快速发现盲点
2. **用户旅程地图** — 找到最容易让用户放弃的那个节点
