# 局部状态覆写模式

## 场景
React 子组件在父组件函数体内定义，需要更新 UI 但调用父 setter 会导致重渲染 -> remount -> local state 丢失。

## 错误做法
子组件 handler 中调用 `setCards(prev => ...)` -> 父重渲染 -> React 为新函数创建新引用 -> remount -> `menuCardId` 归 null。

## 正确做法
子组件维护覆写层：
- `starOverrides` Record -- 乐观更新的状态覆写
- `hiddenIds` Set -- 删除操作的隐藏列表
- 渲染时优先读覆写值
- 操作更新覆写 + fire-and-forget API
- API 失败时回滚覆写

父 `setCards` 仅用于：首次加载、外部数据变更（首页 vs 收藏页切换）。

## 核心原则
1. 子组件本地 UI 状态不依赖父组件 setter
2. 乐观更新通过本地覆写层实现
3. `visibleCards` = props 数据 filter 覆写，不从父 setter 中 splice
