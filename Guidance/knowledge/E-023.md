# 意图分类器误判为"其他" (三层根因)

## 现象
多个对话被意图分类器判定为"其他"，实际明显属于概念理解/技能学习等具体类别。

## 根因 (三层)
1. Prompt ~300 行 3500 tokens -> LLM 偷懒，倾向于选 "other"
2. `intent_by_key()` 只匹配英文 key -> LLM 偶尔输出中文值，fallback 到 "其他"
3. **最关键**: few-shot 示例放在 `## 示例输出` 之下，`extract_prompt_block` 提取范围不含此节（提取到 `## 示例输出` 为止），13 个 few-shot 示例从未发送给 LLM

## 修复
1. Prompt 从 300 行压缩到 ~110 行决策树格式，按优先级 1->N 逐条匹配，每条 1 行 + 关键词
2. `intent_by_key()` 新增中文标签反向匹配（先英文 key -> 再中文反向查找 -> 兜底值）
3. few-shot 移到 `## 典型范例` 节（在 `## 示例输出` 之前）

## 教训
Prompt 辅助函数（extract_prompt_block）的提取边界必须与 Prompt 文件结构严格对齐。few-shot 放在提取范围之外 = 白写 = LLM 收不到。正确的布局：`## 角色设定` -> `## 典型范例`(few-shot) -> `## 输出格式` -> `## 示例输出`。
