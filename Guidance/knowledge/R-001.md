# 本地测试启动方式 (R-001)

- **场景**: 需要启动 Memora Tauri 客户端进行本地测试
- **做法**: 在项目根目录运行 `npx tauri dev`，它会自动：
  1. 启动 Vite 开发服务器（`beforeDevCommand`: `npm run dev`，端口 5173）
  2. 编译并启动 Rust 后端（端口 17321）
  3. 打开 Tauri 桌面窗口（连接 Vite dev server，加载最新源码）
- **为什么**:
  - `cargo tauri` 子命令可能未安装（需 `cargo install tauri-cli`），但 `npx tauri` 通过 npm 包 `@tauri-apps/cli` 始终可用
  - **不要直接运行 `src-tauri/target/debug/llm-knowledge-base.exe`**：debug 构建是 console subsystem，会附着终端窗口，关闭终端会杀死应用；且 debug exe 可能使用打包的 dist 而非 Vite 实时源码
  - `npx tauri dev` 构建的是 Windows GUI subsystem，不会显示终端窗口
- **关联**: [[P-011]] — 启动后可在客户端验证 Logo 等视觉变更
