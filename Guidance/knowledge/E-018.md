# FTS 触发器导致数据库损坏

## 现象
FTS 触发器执行时 SQLite 数据库静默损坏。

## 根因
`content='knowledge_cards', content_rowid='rowid'` 与 TEXT PRIMARY KEY 表不兼容。FTS content 表模式要求 rowid 为整数，但 knowledge_cards 主键是 TEXT 类型。

## 修复
改为独立 FTS 表 + `card_id` 字符串关联：
- `cards_fts` 独立表，通过 `card_id` 列关联 knowledge_cards
- 不再使用 content 外部表模式
- UPDATE 触发器改为显式操作独立 FTS 表

## 教训
SQLite FTS5 content 外部表模式强依赖 rowid 为整数。TEXT PRIMARY KEY 的表必须使用独立 FTS 表模式。
