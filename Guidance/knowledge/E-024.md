# 列表页时间与详情页不一致 + Tauri 外部链接无反应

## 现象
1. 列表页显示时间和详情页不同
2. Tauri HTTP mode 下外部链接点击无反应

## 根因
1. 列表用 `created_at`(卡片创建时间)，详情用 `captured_at`(对话抓取时间)，两个字段含义不同
2. Tauri HTTP mode 下页面走 `http://localhost`，不注入 `__TAURI_INTERNALS__`，前端 IPC 不可用

## 修复
1. 统一用 `source.captured_at` 作为展示时间
2. 后端新增 `POST /api/open-url` 端点（Rust 用 `open::that`，Express 用 `child_process.exec`）

## 教训
时间字段语义必须全链路统一；IPC 能力在 HTTP mode 下不可用，需后端提供替代方案。
