# Windows ICO 多尺寸 + Tauri 窗口图标设置 (P-013)

- **场景**: 
  1. 生成 Windows 桌面应用图标（.ico）时，图标在部分分辨率/DPI 下模糊
  2. Tauri 2.0 运行时窗口（标题栏+任务栏）显示默认图标而非产品 Logo
- **做法**:
  1. **ICO 全尺寸覆盖**：`icon.ico` 必须包含 `[16, 24, 32, 48, 64, 96, 128, 256]` 全部八档尺寸。Windows 在不同 DPI 缩放（100%/125%/150%/200%）和不同场景（任务栏/文件资源管理器/开始菜单）会使用不同尺寸，缺失的尺寸会被放大导致模糊
  2. **Tauri 窗口图标**：在 `main.rs` 的 `setup` 闭包中，使用 `include_bytes!` 在编译时嵌入 PNG 图标，通过 `tauri::image::Image::from_bytes()` 加载，调用 `window.set_icon()` 设置
  3. **需要 `image-png` feature**：Cargo.toml 中 `tauri = { features = ["image-png"] }`，否则 `Image::from_bytes()` 不可用
  4. **Tauri PNG 应覆盖常用 DPI**：`32x32.png`、`64x64.png`、`128x128.png`、`128x128@2x.png`（256px），在 `tauri.conf.json` 的 `bundle.icon` 中全部列出
- **原因**: `tauri.conf.json` 的 `bundle.icon` 只在 `tauri build` 打包时生效（嵌入安装包）；开发模式/运行时窗口图标需要 Rust 侧显式设置。ICO 是容器格式，内含多张 PNG，Windows Shell 按需取用——缺尺寸 = 强制缩放 = 模糊
- **注意**: 用 256px（`128x128@2x`）的 PNG 做窗口图标源最合适，能覆盖 4K 显示器。`include_bytes!` 将图标数据编译进二进制，零运行时文件依赖
