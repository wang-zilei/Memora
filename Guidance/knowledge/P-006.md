# topic 与 tags 不冗余设计

## 场景
两个字段语义重叠，维护成本翻倍，用户困惑。

## 错误做法
同时保留 topic (单一扁平字符串) 和 tags (层级+多值)。

## 正确做法
保留更灵活的字段，删除冗余的：
- tags 支持层级格式 (父/子) 和多值 -> 保留
- topic 是单一扁平字符串，信息量完全被 tags 覆盖 -> 删除

全链路清除：数据库 schema + 前后端类型 + API + 全部 10 个 prompt。

## 核心原则
当两个字段语义重叠时，问一个问题："哪个字段的信息量是另一个的真子集？" 删除子集。
