# Tauri 运行时窗口显示默认图标 (E-029)

- **日期**: 2026-06-04
- **症状**: 桌面快捷方式图标清晰（`.ico` 直接生效），但运行中任务栏图标模糊/显示 Tauri 默认齿轮而非 Memora Logo
- **根因**: 两重问题：
  1. Rust 代码从未调用 `window.set_icon()`——`tauri.conf.json` 中 `bundle.icon` 只在打包安装包时嵌入，**运行时窗口图标需要单独在 Rust 侧设置**
  2. `icon.ico` 只包含 4 个尺寸（32/64/128/256），缺少 Windows 任务栏在不同 DPI 下需要的 16/24/48/96 档位，系统被迫拿 32 放大到 48 → 模糊
- **修复**:
  1. `Cargo.toml`：tauri 加 `image-png` feature，使 `Image::from_bytes()` 可用
  2. `main.rs` setup 闭包：`include_bytes!("../icons/128x128@2x.png")` + `window.set_icon(icon)`
  3. `scripts/convert-icon.cjs`：ICO 尺寸从 `[32,64,128,256]` 扩展为 `[16,24,32,48,64,96,128,256]` 八档；Tauri PNG 增加 64x64
  4. `tauri.conf.json`：`bundle.icon` 增加 `icons/64x64.png`
- **教训**: Tauri 2.0 的图标有两层——`bundle.icon`（打包时）和 `window.set_icon()`（运行时），两者互不替代；Windows ICO 必须覆盖 16/24/32/48/64/96/128/256 全尺寸才能在所有 DPI 下清晰
