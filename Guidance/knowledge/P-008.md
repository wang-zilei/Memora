# Prompt few-shot 示例必须在提取范围内

## 场景
使用 `extract_prompt_block` 从 markdown 文件中提取 prompt 内容发送给 LLM。

## 错误做法
Few-shot 示例放在 `## 示例输出` 之后 -> 被 `extract_prompt_block` 丢弃 -> LLM 从未收到 -> 白写。

## 正确做法
Prompt 文件布局：
1. `## 角色设定` -- 角色和能力描述
2. `## 典型范例` -- few-shot 示例
3. `## 输出格式` -- JSON 格式约束
4. `## 示例输出` -- 输出示例（提取截断点）

提取范围始终是 `## 角色设定` -> `## 示例输出`（不含）。

## 核心原则
了解 prompt 辅助函数的提取边界。放在边界之外的内容不会发送给 LLM。
