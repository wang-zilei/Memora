# 话题拆分过细 + 去重太弱

## 现象
4 条 user 消息的对话生成 13-30 张近似卡片，大量内容重复。

## 根因
1. LLM 把同一主题下的子话题拆成独立块（"不同讨论角度"被当作"话题转变"）
2. 去重只看标题/问题 Jaccard 相似度，粒度太粗

## 修复
1. Prompt 新增"合并优先"、"同主题子话题不拆分"原则
2. 去重从 5 条扩展到 7 条综合判断规则：
   - 核心：original_question Jaccard >= 0.7 + 标题相似度 >= 0.3
   - 辅助：标题 Jaccard >= 0.6 + 问题相似度 >= 0.3
   - 兜底：标题包含 + 问题包含；narrative 前 200 字重度重叠 >= 0.65

## 教训
话题切分不可靠（LLM 容易把不同讨论角度当话题转变），必须搭配强去重引擎。见 P-005。
