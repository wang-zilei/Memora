# React 内联组件 remount 导致 local state 丢失

## 现象
点击收藏或删除按钮后，弹出菜单瞬间消失，操作无反馈。`flushSync` 和 fire-and-forget 均无法解决。

## 根因
子组件定义在父组件函数体内，handler 调用父组件 `setCards` 触发父重渲染 -> React 为内联组件创建新函数引用 -> reconciler 视为不同组件 -> remount -> `menuCardId` 等 local state 归 null。问题在组件 identity 而非渲染时序。

## 修复
局部状态覆写模式 (P-010)：子组件维护 `starOverrides` Record + `hiddenIds` Set，渲染时优先读覆写值，不调用父 setter。父 setCards 仅用于首次加载、外部数据变更。

## 教训
当 flushSync 和 fire-and-forget 均无效时，问题在 React 组件 identity 而非渲染时序。
