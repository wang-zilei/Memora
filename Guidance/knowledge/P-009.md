# LLM 分类输出多语言兼容 (双语 key 匹配)

## 场景
LLM 可能自作主张输出中文而非 prompt 要求的英文 key。

## 错误做法
只匹配英文 key，LLM 输出中文值 -> 匹配失败 -> fallback 到默认值。

## 正确做法
两步匹配：
1. 先尝试英文 key 直接匹配
2. 如果失败，用中文值反向查找对应 key
3. 如果仍失败，返回兜底值

规范化后统一用英文 key 做下游路由。

## 核心原则
永远不假设 LLM 会严格遵守输出语言要求。英文 key + 中文反向映射 = 双语保险。
