# Tauri 完全缺少对话清洗逻辑

## 现象
Tauri 客户端数据链路全错：raw_json 同时绑定到 raw 和 clean 表，AI Pipeline 接收原始消息（含 `<think>` 标签、平台垃圾），卡片质量极差。

## 根因
Rust 端没有移植 Demo 的对话清洗模块。`raw_json` 被直接写入 clean 表，跳过了清洗步骤。

## 修复
新增 4 个清洗函数完整移植到 Rust：
1. `normalize_role()` — 角色名标准化
2. `clean_content()` — 去除 think 标签和平台垃圾
3. `merge_consecutive()` — 合并连续同角色消息
4. `clean_conversation()` — 完整清洗管线

数据流改为：先清洗 -> 存储到 clean 表 -> 传入 AI Pipeline。

## 教训
双端（Demo Express + Tauri Rust）共存的架构中，每端必须独立验证数据处理流程的完整性。一个环节缺失会导致全链路数据质量崩溃。
