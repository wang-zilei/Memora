## 白色 ins 风最终收口

**反馈**:

- 核心问题渐变偏鲜艳，需要换成更柔和的色系。
- 整体白色 UI 仍有灰蒙感，需要更清爽。
- 搜索、总数、刷新区域存在感偏强，希望淡化并合并到一行。

**修复**:

- 背景和主区域提亮为更清透的近白底，减少灰度感。
- 首页控制区调整为 `搜索 + 搜索按钮 + 总数 + 刷新` 同一行，状态按钮使用更淡的边框和文字。
- 核心问题背景改为极轻的暖白到清透蓝白渐变，降低蓝粉饱和度。

**验证**:

- `npm run build` 通过。

- TipTap toolbar 改用 Material Symbols 的 `format_bold`、`format_italic`、`format_underlined`、`border_color` 图标，并统一按钮尺寸。

## 视觉反馈二次调整

**反馈**:

- 当前配色不满意，需要换一套 color token 并去掉背景网格。
- 列表卡片虽然恢复 3 列，但卡片比例偏长、质感不足，边框弧度观感僵硬。
- 意图标签不应统一颜色，需要恢复不同意图对应不同颜色。
- 详情页标题过大、间距过大，整体观感差。
- card_type 自定义选择标签打开时 chip 变形。

**修复**:

- 页面和主内容背景改为纯净浅暖底，移除网格背景。
- token 调整为更低饱和的暖灰、纸白、墨色、sage/rose 辅色体系。
- 列表卡片增加纵向高度、微渐变、内高光和分层阴影，维持 3 列但减少横向扁长感。
- 恢复并重设 10 个意图的独立低饱和色映射，移除覆盖 inline 意图色的 `!important`。
- 详情页压缩 header、title、tabs、section、TipTap 的字号和间距。
- card_type 选择 chip 改为可换行、固定轻量尺寸的正常 chip，避免横向滚动和形变。

**验证**:

- `npm run build` 通过。

- 详情页恢复标题下分隔线、详情卡外边框、核心问题/关键结论字号与 TipTap 编辑器边框。
